# Web Scraping Glossary

> A reference guide to common terms and concepts in the web scraping industry.

## A

**API (Application Programming Interface)**
A set of protocols and tools that allows different software applications to communicate with each other.

**ASP (Application Service Provider)**
A third-party provider that hosts applications over the internet.

## C

**Crawl**
To systematically browse and index web pages, typically for the purpose of building a search index or discovering content.

**CSS Selector**
A pattern used to select and target specific HTML elements on a page for data extraction.

## H

**Headless Browser**
A browser without a graphical user interface that can render and interact with web pages programmatically.

## P

**Pagination**
The practice of dividing content across multiple pages, requiring scrapers to navigate through multiple URLs or interactions.

**Proxy**
An intermediary server that routes your requests, masking your original IP address to avoid detection.

## R

**Render**
To execute JavaScript and produce the final HTML output that a real browser would display.

## S

**Scraping**
The automated collection of data from websites using software tools.

**Selectors**
Patterns (XPath, CSS) used to target specific elements on a page for extraction.

## X

**XPath**
A language for selecting nodes from an XML document, commonly used in web scraping to target HTML elements.

## 何时使用 API Key

| 认证方式 | 适用场景 |
| --- | --- |
| **OAuth 2.1** | ChatGPT、Claude、Cursor、Gemini 等支持浏览器登录的客户端；Coze、Dify 安装官方工具时 |
| **API Key** | Coze 自定义 MCP、Dify API Key 模式、QClaw、Cursor、Claude Code、脚本 / 无头环境 |

## 创建 API Key

<Steps>
  <Step title="登录并打开账号信息">
    登录 [https://www.bazhuayu.com/](https://www.bazhuayu.com/) 后点击账号信息。

    <img src="/assets/mcp/api-key/step-1.png" alt="登录八爪鱼官网后点击账号信息" className="mcp-step-image" />
  </Step>
  <Step title="进入 API 密钥并新增">
    点击 API 密钥，新增密钥。（建议每个连接的智能体都有独立的密钥便于管理）

    <img src="/assets/mcp/api-key/step-2.png" alt="在账户中心点击 API 密钥并新增密钥" className="mcp-step-image" />
  </Step>
  <Step title="自定义名称并生成">
    自定义密钥名称，点击生成。

    <img src="/assets/mcp/api-key/step-3.png" alt="填写密钥名称并点击生成密钥" className="mcp-step-image" />
  </Step>
  <Step title="复制并妥善保存密钥">
    此后便获取到密钥。（注意！！！密钥只会在这个界面显示一次，后续无法获取，一定妥善保存。同时建议每次连接新载体都新建密钥，便于管理）

    <img src="/assets/mcp/api-key/step-4.png" alt="生成后复制 API 密钥并妥善保存" className="mcp-step-image" />
  </Step>
</Steps>

<Warning>
  不要将 API Key 提交到 Git、飞书文档公开页、截图或客服群。仅在 MCP 客户端的「密钥 / Headers」配置项中填写。
</Warning>

## 在 MCP 客户端中配置

### 通用 HTTP MCP 配置

```json
{
  "type": "http",
  "url": "https://mcp.bazhuayu.com",
  "headers": {
    "x-api-key": "你的APIKey"
  }
}
```

### Cursor 示例（`~/.cursor/mcp.json`）

```json
{
  "mcpServers": {
    "octoparse": {
      "type": "http",
      "url": "https://mcp.bazhuayu.com",
      "headers": {
        "x-api-key": "你的APIKey"
      }
    }
  }
}
```

### Claude Code 示例（`~/.claude/settings.json`）

```json
{
  "mcpServers": {
    "octoparse": {
      "type": "http",
      "url": "https://mcp.bazhuayu.com",
      "headers": {
        "x-api-key": "你的APIKey"
      }
    }
  }
}
```

保存后重启客户端，在 MCP 工具列表中应能看到八爪鱼相关工具。

更多客户端（ChatGPT、Gemini、OpenClaw、VS Code）见 [客户端配置指南](/zh/mcp/guides/clients)。

## 验证是否生效

连接成功后，可在对话中尝试：

- 「用八爪鱼搜索一个采集新闻标题的模板」
- 「列出我账户下的八爪鱼任务」

若返回 `401` / `403`，请检查 Key 是否有效、是否填在 `x-api-key` 字段，以及 URL 是否正确。

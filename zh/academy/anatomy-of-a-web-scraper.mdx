# 网页采集器的结构解析

> 每个网页采集器都遵循相同流程：请求、渲染、定位、提取、导航和存储。

## 采集流程

<Steps>
  <Step title="请求">
    向目标 URL 发送 HTTP 请求并接收页面 HTML。关键决策：标准 HTTP 客户端还是真实浏览器、合适的请求头、超时处理。
  </Step>
  <Step title="渲染">
    执行 JavaScript 以展示动态内容。可选方案：无头浏览器、API 拦截、SSR 检测。
  </Step>
  <Step title="定位">
    识别哪些 HTML 元素包含目标数据。方法：CSS 选择器、XPath、AI 驱动元素检测、正则表达式。
  </Step>
  <Step title="提取">
    取出实际数据 — 文本内容、属性、表格、价格、日期。
  </Step>
  <Step title="导航">
    处理翻页（页码翻页、下一页按钮、无限滚动）、点击进入详情页、表单提交、项目遍历。
  </Step>
  <Step title="存储">
    保存至 CSV/Excel、JSON、Google Sheets、数据库（MySQL、PostgreSQL、MongoDB）或云存储。
  </Step>
</Steps>

# XPath 与 CSS 选择器

> CSS 选择器和 XPath 都能在页面上定位元素，但它们的失效方式不同。了解各自的原理、何时选用哪种，以及如何编写持久可靠的选择器。

## CSS 选择器

CSS 选择器是定位元素的主要方式。它们被广泛支持且可读性好。

**常用 CSS 选择器：**
```
/* 按标签 */
div

/* 按 class（前缀 .） */
.product-card

/* 按 ID（前缀 #） */
#main-content

/* 按属性 */
[data-product-id="123"]

/* 后代选择器 */
.container .product-title

/* 直接子元素 */
ul > li

/* 多 class */
.item-price.sale.final
```

## XPath

XPath 比 CSS 选择器功能更强大，尤其适用于向上导航、按文本内容选择和构建复杂条件。

**常用 XPath 表达式：**
```
//div[@class="product"]                     # class 为 "product" 的 div
//a[contains(@href, "category")]           # href 中含 "category" 的链接
//h1[text()="产品名称"]                     # 精确文本匹配的 h1
//section[starts-with(@id, "section")]     # id 以 "section" 开头的 section
//..                                        # 父元素
```

## CSS vs XPath

| 特性 | CSS 选择器 | XPath |
|---|---|---|
| 语法 | 简洁易读 | 较为冗长 |
| 向上遍历 | 不支持 | 支持（.., ancestor::） |
| 文本匹配 | 有限 | 完整 text() 支持 |
| 位置选择 | :nth-child() | [position()] |
| 浏览器支持 | 通用 | 通用 |
| 八爪鱼 中使用 | 推荐 | 可用 |

## 编写持久选择器

网站结构变化时选择器会失效。提高持久性的技巧：

1. **使用语义化属性** — 优先使用 data-*、aria-* 或有意义的 ID
2. **避免位置相关的选择器** — nth-child 在添加新元素时会失效
3. **使用最短的唯一路径** — 过长则脆弱，过短则歧义
4. **优先使用 class 和 ID** — 结构选择器如 div:nth-child(3) > span 极易失效
5. **在多个页面上测试** — 在一个页面上有效的选择器可能在另一个页面上失效

# 如何采集 Amazon 数据

> 提取 Amazon 产品数据的实用指南：搜索结果、产品页面、评论、变体、价格、卖家数据及反爬考量。

Amazon 是最有价值但也是最难采集的电商数据源之一。产品页面包含价格、卖家、评论、变体、排名、图片和规格，但网站是动态的、被严密监控的，且不同站点的布局可能不同。

## 页面类型

| 页面类型 | 最佳用途 | 典型字段 |
|---|---|---|
| 搜索结果 | 产品发现与排名 | 标题、URL、图片、价格、评分、评论数、ASIN |
| 分类页面 | 品类研究 | 产品卡片、排名位置、分类 URL |
| 产品详情页 | 深度产品情报 | 品牌、ASIN、价格、卖家、卖点、描述、规格、变体 |
| 评论页面 | 情感与质量分析 | 评论标题、评分、正文、日期、评论者、已验证购买 |
| 卖家页面 | 市场监控 | 卖家名称、店铺 URL、卖家评分 |

## 第一步：选择输入

需要产品发现时使用**关键词**：
```
无线键盘
蛋白粉
儿童冬靴
```

已知产品目录时使用**产品 URL 或 ASIN**：
```
https://www.amazon.com/dp/B08...
B08...
```

对于价格监控，ASIN 或产品 URL 通常优于重复的关键词搜索。

## 第二步：采集列表数据

搜索结果和分类页面适合广度覆盖。采集：
- 产品标题
- 产品 URL 和 ASIN
- 价格
- 评分和评论数
- 图片 URL
- 广告/自然结果标识
- 结果排名
- 时间戳

<Note>
  结果排名很重要。产品在某个关键词下的排名可能与其价格同样关键。
</Note>

## 第三步：访问产品详情页

详情页提供深度分析所需字段：
- 品牌和功能卖点
- 产品描述和规格
- 变体
- 卖家和配送信息
- 畅销榜排名
- 完整图片集
- 优惠券或促销信号

## 第四步：单独采集评论

评论提取是一个独立的流程。排序方式、翻页、语言、星级筛选和区域都会影响输出。

有用的评论字段：评分、评论标题、评论正文、日期、评论者名称、已验证购买标识、有用投票数、产品变体。

## 技术挑战

采集 Amazon 数据通常会遇到：
- 动态布局和 A/B 测试
- 不同地区的价格和库存差异
- CAPTCHA 和反爬检测
- 变体特定字段
- 缺失或隐藏的数据

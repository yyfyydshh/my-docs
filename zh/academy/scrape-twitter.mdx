# 如何采集 Twitter/X 数据

> 如何在遵守平台政策和 API 替代方案的前提下，从 Twitter/X 提取公开推文、个人资料、趋势和互动数据。

## 采集内容

| 数据类型 | 字段 |
|---|---|
| 推文 | 文本、作者、日期、点赞数、转发数、回复数、URL |
| 个人资料 | 用户名、显示名称、简介、粉丝数、关注数、位置 |
| 趋势 | 趋势名称、热度、地区 |
| 话题标签 | 标签名称、使用次数、近期推文 |

## 法律与伦理考量

<Warning>
  Twitter/X 有严格的服务条款和强力执法机制。官方 API 是推荐方式。未经授权的采集可能导致法律诉讼或 IP 封禁。在进行之前请查看 Twitter 的自动化规则。
</Warning>

## API vs 采集

Twitter 提供各种访问级别的官方 API。免费层有限制；付费层提供更多数据。对大多数场景而言，官方 API 是正确的选择：
- **免费层** — 每月 1,500 条推文（含用户上下文）
- **基础层** — $100/月，更高级别限制
- **专业层** — 更高限制和全量归档搜索

当 API 不足时，使用真实浏览器采集可以提供额外的数据访问，但承担重大的合规风险。

## 常见应用场景

- **品牌监控** — 追踪品牌和产品的提及和情感
- **竞品分析** — 监控竞品社交活动
- **趋势分析** — 识别新兴话题和讨论模式
- **线索生成** — 找到讨论特定话题或需求的用户

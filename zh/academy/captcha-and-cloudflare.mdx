# 绕过 CAPTCHA 和 Cloudflare

> CAPTCHA 挑战和 Cloudflare 式反爬防护的工作原理、为何在采集过程中出现，以及哪些层面有助于减少或解决它们。

CAPTCHA 和 Cloudflare 挑战是网站不再信任该会话的信号。触发原因可能是 IP 地址、浏览器指纹、请求频率或这些因素的组合。

## CAPTCHA 在测试什么

CAPTCHA 是一个挑战-响应层。网站要求访客做一些对人类更容易的事：
- **图片挑战** — 选择红绿灯、公交车、店面
- **复选框挑战** — 评估浏览器和交互信号
- **隐形风险评分** — 仅在会话看起来可疑时才显示挑战
- **Turnstile 验证** — 基于浏览器的验证，摩擦更少

<Note>
  在数据采集中，CAPTCHA 通常意味着之前某个层面看起来可疑。解决可见的挑战并不能修复糟糕的 IP 或不一致的指纹。
</Note>

## Cloudflare 额外增加什么

Cloudflare 施加多个层面：JavaScript 完整性检查、托管挑战、速率限制、Turnstile 验证，以及基于地区和 IP 信誉的访问规则。

## 分层应对策略

<Steps>
  <Step title="当网站期望真实浏览器时使用真实浏览器">
    如果目标页面依赖 JavaScript 或 Cloudflare 浏览器检查，则原始 HTTP 请求是错误的工具。
  </Step>
  <Step title="保持指纹一致">
    IP 区域、时区、语言、User-Agent 和视口信息应该共同讲述一个连贯的故事。
  </Step>
  <Step title="将请求量与用户模式匹配">
    即使使用优质代理，从一个 IP 段每小时 10,000 个请求看起来也像攻击。
  </Step>
  <Step title="添加类人化行为">
    鼠标曲线、随机点击偏移、可变时间间隔和滚动行为。
  </Step>
  <Step title="仅将 CAPTCHA 解决作为后备手段">
    自动化解决不能修复根本原因。谨慎使用，首先解决根本原因。
  </Step>
</Steps>

# 什么是网页数据采集？

> 网页数据采集是从网站自动提取结构化数据的过程。了解其工作原理、应用领域，以及与 API 和人工收集的区别。

## 概述

网页数据采集使用软件导航网页、识别所需数据，并以 CSV、Excel 或 JSON 等结构化格式保存 — 无需手动复制粘贴。

## 为什么需要网页数据采集？

- **规模化** — 数分钟内采集数千条数据
- **周期性** — 按计划自动定期采集数据
- **结构化** — 将非结构化网页内容转化为整洁的表格数据
- **成本** — 消除重复性人工劳动
- **竞品情报** — 监控价格、产品及市场趋势

## 采集流程

每个网页采集器都遵循以下环节：

1. **请求** — 请求目标 URL 并获取页面 HTML
2. **渲染** — 必要时执行 JavaScript 以展示动态内容
3. **定位** — 找到包含目标数据的特定元素
4. **提取** — 从这些元素中取出数据
5. **导航** — 翻页、点击链接或遍历列表
6. **存储** — 将结果保存至文件、数据库或 API 目标

## 网页采集 vs API

| | 网页采集 | API |
|---|---|---|
| **访问方式** | 任何公开网站 | 仅限开发者 |
| **数据范围** | 所有可见数据 | 受限于 API 开放范围 |
| **搭建** | 需要采集工具或代码 | 直接 HTTP 请求 |
| **维护** | 页面变动则选择器失效 | API 版本管理有帮助 |

## 是否合法？

网页采集的合法性取决于你采集什么、如何采集以及所在地区。关键考量：公开数据与私有数据、服务条款、版权及 GDPR/CCPA。

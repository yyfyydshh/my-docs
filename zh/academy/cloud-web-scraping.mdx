# 云端网页采集详解

> 为何网页采集器要迁移至云端：无人值守运行、子任务并行、定时调度及网络基础设施。

## 本地 vs 云端提取

| | 本地 | 云端 |
|---|---|---|
| **运行环境** | 你的机器 | 八爪鱼 云端服务器 |
| **可用性** | 需要你的电脑开机 | 7×24 无人值守 |
| **可扩展性** | 受硬件限制 | 子任务并行运行 |
| **IP 管理** | 你的 IP 地址 | 轮换住宅/数据中心 IP |
| **反爬处理** | 基础 | 高级代理轮换 |
| **定时调度** | 手动触发 | 内置定时调度 |

## 为何迁移至云端？

1. **无人值守运行** — 无需电脑开机即可定时运行（每小时、每天、每周）
2. **子任务并行** — 将大型任务分为并行子任务，加速提取
3. **更优的反爬规避** — 轮换 IP 池使采集更难被检测
4. **资源释放** — 提取在远程运行时，你的机器保持空闲
5. **可靠性与监控** — 运行历史、错误告警、重试逻辑

## 何时使用云端提取

- 需要定时调度的周期性数据采集
- 大规模提取（数千页面）
- 具有强力反爬措施的目标网站
- 长时间运行的任务
- 复杂的多步骤工作流

# 如何处理翻页

> 了解爬虫如何超越第一页：页码翻页、下一页链接、无限滚动、加载更多按钮、API 偏移量及游标分页。

## 从请求入手，而非 UI

在编写翻页逻辑之前，打开 DevTools 观察什么发生了变化：
1. 打开 Network 标签，筛选 Fetch/XHR
2. 点击下一页、滚动或按下加载更多
3. 检查请求 URL 和响应
4. 决定是跟随链接、与页面交互还是直接调用 API

## 常见模式

| 变化内容 | 首先尝试的方法 |
|---|---|
| URL 包含 page=2 | 循环遍历编号的 URL |
| 锚链接指向下一页 | 跟随 href 直到其消失 |
| 内容在滚动后出现 | 找到 XHR 请求 |
| 内容在点击按钮后出现 | 复用 API 请求 |
| JSON 包含 next_cursor 或 has_more | 通过 API 响应进行分页 |

## 常见陷阱

- **重复第一页** — 某些网站在超出范围时返回第一页，造成无声重复
- **缺少边界检测** — 爬虫在空的最后一页上无限循环
- **会话感知分页** — 会话中途更换 IP 可能重置页数

# 浏览器运行时全景图

> 用于网页采集的浏览器运行时实地指南：Puppeteer、Playwright、Selenium、云端 API 及 八爪鱼。

## 浏览器自动化运行时

### Puppeteer
Google 官方的 Chrome/Chromium 控制 Node.js 库。
- **最适合：** Chrome 专属自动化
- **优点：** 出色的 Chrome 支持，活跃开发
- **缺点：** 仅限 Chrome

### Playwright
Microsoft 的跨浏览器自动化库（Chrome、Firefox、WebKit）。
- **最适合：** 跨浏览器测试、现代采集需求
- **优点：** 多浏览器支持，自动等待机制
- **缺点：** 单浏览器使用时体积较大

### Selenium
最古老且使用最广泛的浏览器自动化框架。
- **最适合：** 遗留项目、Java 为主的环境
- **优点：** 语言无关，庞大社区
- **缺点：** 较慢，设置更复杂

### 云端采集 API
Bright Data、ScraperAPI、Oxylabs 通过 API 提供基于浏览器的采集。
- **最适合：** 避免基础设施管理
- **优点：** 无需管理基础设施，内置代理轮换
- **缺点：** 按请求计费

| 维度 | Puppeteer | Playwright | Selenium | 云端 API | 八爪鱼 |
|---|---|---|---|---|---|
| 需要编码 | 是 | 是 | 是 | 最低 | 否 |
| 反爬适应力 | 中 | 中 | 中 | 高 | 高 |
| 成本 | 免费 | 免费 | 免费 | 按请求 | 订阅制 |

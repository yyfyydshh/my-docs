# 浏览器指纹

> 反爬系统通过 Canvas、WebGL、字体、UA 及数十种其他信号的组合签名来识别爬虫。

每个浏览器会话都会泄露一个识别签名 — User-Agent、Canvas 绘制输出、WebGL 渲染器字符串、字体列表、屏幕分辨率、时区、语言头部等信息的组合。现代反爬系统将这些信号合并为一个**浏览器指纹**，其识别会话的可靠性远超 IP 地址。

指纹层位于行为之下：它描述的是会话"是什么"，而不是"做什么"。爬虫必须通过这两层才能在严格的反爬系统面前保持隐身。

## 追踪哪些信号

- **User-Agent 和 Accept 头部** — UA 中出现 HeadlessChrome 是一个明显的暴露标志
- **Canvas 指纹** — 隐藏的 Canvas 渲染输出为每台机器生成唯一的哈希值
- **WebGL 指纹** — 渲染器字符串、供应商字符串、支持的扩展
- **字体列表** — 浏览器可渲染的字体及其顺序
- **屏幕与视口** — 分辨率、色彩深度、设备像素比
- **时区与语言** — 来自 Intl.DateTimeFormat 和 navigator.languages
- **音频上下文** — 音频渲染产生设备特定的输出
- **navigator.webdriver** — 未管理的自动化操作的致命暴露点

## 为何爬虫会泄露指纹

三种主要失败模式：

1. **重复使用相同默认身份** — 一个原生 Puppeteer 实例运行 10,000 个会话，呈现相同的 Canvas 哈希、相同的 WebGL 字符串 — 10,000 个"不同用户"却共享一台机器的指纹
2. **使用真实用户不会产生的通用值** — 空的 navigator.plugins、精确匹配的标准 Linux/Chrome Canvas 输出
3. **信号不匹配** — en-US 和 America/New_York 却搭配俄罗斯 IP

## 管理指纹

<Steps>
  <Step title="每会话唯一性">
    每个任务实例应呈现不同的指纹 — 不同的 Canvas 哈希、不同的 WebGL 渲染器。
  </Step>
  <Step title="会话内一致性">
    在一个会话中，指纹必须保持稳定；中途切换本身就是暴露信号。
  </Step>
  <Step title="地理一致性">
    指纹的时区、语言和头部应与代理 IP 的地理位置匹配。
  </Step>
  <Step title="真实而非随机">
    由纯粹随机值拼凑的指纹是异常的。应从真实世界指纹的分布中采样。
  </Step>
</Steps>

# 网页采集是否合法？

> 网页采集的合法性取决于你采集什么、如何采集以及所在地区。概述公开数据、服务条款、版权、GDPR/CCPA 以及如何合规采集。

## 关键法律考量

### 1. 公开数据 vs 私有数据

**公开数据**（无需登录即可见）通常风险较低。私有数据（需登录或付费墙后）则引入额外的法律问题。

### 2. 服务条款

大多数网站在其服务条款中禁止数据采集。不同司法管辖区的法院得出了不同的结论。

### 3. 版权

网站的架构和创意表达可能受版权保护。事实不受版权保护 — 采集价格和联系方式通常较为安全。

### 4. GDPR 与 CCPA

采集欧盟或加州居民的个人数据可能使你成为数据控制者并承担相应义务。

## 实用风险缓解措施

<Steps>
  <Step title="尽量采集公开数据">
    优先选择公开可访问的页面，而非需要认证的页面。
  </Step>
  <Step title="仔细阅读服务条款">
    在采集前了解你所同意的内容。
  </Step>
  <Step title="遵守 robots.txt">
    即使并非在所有地区都具有法律约束力，也能体现善意。
  </Step>
  <Step title="限制请求频率">
    不要给目标服务器造成过载。
  </Step>
  <Step title="最小化数据收集">
    仅采集你实际需要的数据。
  </Step>
  <Step title="咨询律师">
    对于商业或高风险项目，请获取法律建议。
  </Step>
</Steps>

| 因素 | 风险等级 |
|---|---|
| 公开事实数据 | 低 |
| 公开内容（大规模） | 中 |
| 服务条款禁止的采集 | 中高 |
| 私有/认证数据 | 高 |
| 个人数据（GDPR/CCPA） | 高 |

_以上为一般法律信息，不构成法律建议。请咨询合格律师。_

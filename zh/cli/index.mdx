> 使用八爪鱼 CLI，在命令行中运行、管理八爪鱼采集任务并导出数据。

八爪鱼 CLI 是面向开发者与运维场景的命令行工具。与桌面客户端不同，它更适合**脚本化工作流**、**CI/CD 流水线**、**服务器环境**以及**自动化数据管道**。

## 什么是八爪鱼 CLI？

### 一个简单比喻

日常使用八爪鱼时，你通过客户端或网页**点击按钮**来启动采集；这叫做图形界面，直观但重复操作多，难以批量自动化。

**CLI 把「点击」变成「敲命令」：**

| 方式 | 你怎么操作 | 八爪鱼如何响应 |
| --- | --- | --- |
| 图形界面（客户端 / 网页） | 手动点击 | 按界面操作执行 |
| 命令行（CLI） | 在终端输入命令 | 按命令执行，可写脚本批量跑 |

## 工作原理

- **本地采集**：CLI 直接运行内置八爪鱼引擎，使用独立 Chrome，不依赖桌面客户端界面。
- **云采集**：通过后端 API 控制云端任务启停与状态查询。
- **状态说明**：本地运行状态由 CLI 自行维护，**不会**与桌面客户端中的运行状态同步。

## 环境要求

开始使用前，请确认：

- **Node.js** 20 或更高版本（推荐 22 LTS）
- **npm** 8 或更高版本
- 有效的 [**八爪鱼 API Key**](/zh/mcp/quick-start/api-key)（[创建 API Key](/zh/mcp/quick-start/api-key)）
- 对需要运行或导出的任务具备访问权限

详细安装步骤见 [安装与验证](/zh/cli/quick-start/installation)。

<Note>
  除 `--help`、`--version`、`doctor`、`browser doctor`、`capabilities`、`auth` 等诊断类命令外，**所有功能命令均需 API Key**，包括本地 `--task-file` / `.otd` 运行。请勿将 API Key 提交到 Git、脚本、文档截图或 CI 日志中。
</Note>

## 常见工作流

<Steps>
  <Step title="安装 CLI">
    安装 Node.js 后，通过 npm 全局安装八爪鱼 CLI。
  </Step>
  <Step title="登录认证">
    [创建 API Key](/zh/mcp/quick-start/api-key) 后，使用 API Key 执行 `octopus auth login`，或配置环境变量。
  </Step>
  <Step title="查找任务">
    使用 `octopus task list` 获取任务 ID。
  </Step>
  <Step title="运行或控制">
    本地运行、启动 / 停止云采集，或查看运行状态。
  </Step>
  <Step title="导出数据">
    从本地或云端结果导出结构化数据。
  </Step>
</Steps>

## 快速命令示例

将 `<taskId>` 替换为实际任务 ID；若不知道 ID，先执行 `octopus task list`。

```bash
octopus --help
octopus doctor
octopus auth login
octopus task list
octopus task inspect <taskId>
octopus run <taskId>
octopus cloud start <taskId>
octopus local status <taskId>
octopus data export <taskId> --source local --format xlsx
```

## CLI、MCP 与桌面客户端怎么选？

| 场景 | 推荐方式 |
| --- | --- |
| 在 Cursor、ChatGPT、Claude、Gemini 等 AI 里用自然语言操作八爪鱼 | [MCP 服务](/zh/mcp/index) |
| 写脚本、定时任务、服务器 / CI 自动化 | **八爪鱼 CLI**（本模块） |
| 可视化搭建流程、日常交互式采集 | 八爪鱼桌面客户端 |

CLI 与 MCP 可配合使用：例如在 AI 中通过 MCP 找到任务 ID，再在终端用 CLI 做批量运行与导出。

## 当前限制（v1）

八爪鱼 CLI v1 **不支持**内核浏览器（Kernel browser）及旧版工作流（Legacy workflow）。

若任务不兼容，请在当前版八爪鱼桌面客户端中更新任务定义后，再使用 CLI 运行。

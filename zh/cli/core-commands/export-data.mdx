> 采集完成后，把结果导出成 Excel、CSV 等文件。

## `octopus data export`（把采集到的数据导出成文件）

**什么意思？** 采集完成后，数据保存在 CLI 的本地或云端结果中。此命令将其导出为 Excel、CSV 等文件，方便查看与后续处理。

**基本用法：**

```bash
# 导出本地运行结果为 Excel
octopus data export <taskId> --source local --format xlsx

# 导出云端运行结果为 CSV
octopus data export <taskId> --source cloud --format csv

# 指定输出文件路径
octopus data export <taskId> --source local --format csv --file ./result.csv
```

若本地运行时使用了 `--output ./runs`，导出时需保持一致：

```bash
octopus run <taskId> --output ./runs
octopus data export <taskId> --source local --output ./runs --format xlsx
```

## 如何导出为不同格式？

```bash
octopus data export <taskId> --source local --format xlsx   # Excel（默认，最常用）
octopus data export <taskId> --source local --format csv    # CSV
octopus data export <taskId> --source local --format json   # JSON（适合程序处理）
octopus data export <taskId> --source local --format html   # HTML（浏览器直接打开）
octopus data export <taskId> --source local --format xml    # XML
```

## 各种格式怎么选？

| 格式 | 适合谁 | 优点 | 缺点 |
| --- | --- | --- | --- |
| xlsx（Excel） | 所有人 | 能用 Excel 打开，带格式，最直观 | 文件稍大 |
| csv | 数据分析人员 | 体积最小，各软件通用 | 无格式（粗体、颜色等） |
| json | 开发者 / 脚本 | 结构化，便于程序解析 | 需用编辑器或代码查看 |
| html | 快速预览 | 浏览器直接打开 | 不适合大规模二次处理 |
| xml | 对接老系统 | 通用交换格式 | 文件较大 |

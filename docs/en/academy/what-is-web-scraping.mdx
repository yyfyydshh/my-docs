# What is Web Scraping?

> Web scraping is the automated extraction of structured data from websites. Learn how it works, where it is used, and how it differs from APIs and manual collection.

## Overview

Web scraping uses software to navigate web pages, identify the data you need, and save it in a structured format such as CSV, Excel, or JSON - without manual copying.

## Why Web Scraping?

- **Scale** - Collect thousands of data points in minutes
- **Recurrence** - Automate regular data collection on a schedule
- **Structure** - Convert unstructured web content into clean, tabular data
- **Cost** - Eliminate manual labor for repetitive tasks
- **Competitive Intelligence** - Monitor prices, products, and market trends

## The Pipeline

Every web scraper follows these stages:

1. **Fetch** - Request the target URL and retrieve the page HTML
2. **Render** - Execute JavaScript if needed to display dynamic content
3. **Locate** - Find the specific elements containing the target data
4. **Extract** - Pull the data from those elements
5. **Navigate** - Follow pagination, click links, or loop through listings
6. **Store** - Save results to file, database, or API destination

## Web Scraping vs APIs

| | Web Scraping | APIs |
|---|---|---|
| **Access** | Any public website | Developer-only |
| **Data Scope** | All visible data | Limited to API exposure |
| **Setup** | Requires scraping tool or code | Direct HTTP requests |
| **Maintenance** | Pages change, selectors break | API versioning helps |

## Is It Legal?

Web scraping legality depends on what you scrape, how you scrape it, and where you are. Key considerations: public vs. private data, ToS, copyright, and GDPR/CCPA.
